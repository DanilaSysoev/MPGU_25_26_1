# Задачи 2. Уровень «Senior» (Мини-проекты с файлами)

**Внимание!** Проекты проверяются автоматически. Отклонение от названий файлов или сигнатур функций приведет к невозможности запустить тесты и оценке 0 баллов за автопроверку.

## ❗️ Правила оформления и защиты (КРИТИЧНО)

1. **Изоляция кода:** Весь код, который запускает программу (вызовы `input()`, `print()`, создание файлов для проверки), должен находиться **СТРОГО** внутри блока `if __name__ == "__main__":`. Тестирующая система будет импортировать ваши функции. Если при импорте ваша программа начнет требовать ввод с клавиатуры — тест упадет.
2. **Аддитивная система (10 баллов):** Проект разбит на 4 функции. Баллы начисляются за каждую функцию, которая успешно прошла автоматические тесты (распределение баллов указано в каждой задаче).
3. **Защита работы и Штрафы:** Автоматический балл — это *предварительная* оценка. Итоговая оценка выставляется преподавателем после мини-интервью (защиты).
   * **Штраф до -5 баллов:** «Грязный» код (отсутствие type hints, плохие имена переменных типа `f`, `d`, `lst`, дублирование кода, неиспользование менеджера контекста `with open(...)`).
   * **Аннулирование (0 баллов):** Студент не может объяснить, как работает написанный им код (подозрение на слепое копирование от ИИ или одногруппника). Вы должны понимать каждую строчку, которую сдаете.

## Вариант 1. Анализатор логов сервера

**Файл решения:** `project_logs.py`

**Сценарий:** У вас есть текстовый файл `server.txt`, где каждая строка — это запись о доступе к серверу в формате `IP-адрес - Статус_кода`. Нужно прочитать этот лог, собрать статистику и сохранить её в JSON. Пример строки: `192.168.1.1 - 200`.

**Обязательные функции для реализации:**

1. `read_logs(file_path: str) -> list[str]` **(2 балла)**
   * Читает текстовый файл по пути `file_path` и возвращает список строк (очищенных от символа переноса строки `\n`). Если файла нет, возвращает пустой список.
2. `parse_log_line(line: str) -> dict` **(3 балла)**
   * Принимает одну строку лога. Возвращает словарь вида `{"ip": "192.168.1.1", "status": 200}`. Обратите внимание, что статус должен стать числом (`int`).
3. `calculate_statistics(logs: list[dict]) -> dict` **(3 балла)**
   * Принимает список разобранных логов. Возвращает словарь со статистикой: сколько раз встречался каждый статус-код. Пример: `{"200": 15, "404": 3, "500": 1}`. (Ключи статусов в JSON должны быть строками).
4. `save_to_json(data: dict, file_path: str) -> None` **(2 балла)**
   * Сохраняет переданный словарь `data` в файл `file_path` в формате JSON с отступами (indent=4).

## Вариант 2. Консольная Викторина (Quiz)

**Файл решения:** `project_quiz.py`

**Сценарий:** Игра загружает вопросы из JSON-файла, задает их пользователю, а рекорды лучших игроков сохраняет в текстовый файл `leaderboard.txt`.

**Обязательные функции для реализации:**

1. `load_questions(file_path: str) -> list[dict]` **(2 балла)**
   * Читает JSON-файл, содержащий список словарей (вопросов). Пример структуры: `[{"question": "2+2?", "answer": "4"}]`. Возвращает этот список.
2. `check_answer(question: dict, user_answer: str) -> bool` **(2 балла)**
   * Сравнивает ответ пользователя `user_answer` с правильным ответом в словаре `question`. Проверка не должна зависеть от регистра и лишних пробелов по краям.
3. `save_score(name: str, score: int, file_path: str) -> None` **(3 балла)**
   * Дописывает (режим `a`) в текстовый файл `file_path` строку вида `Имя: Очки\n` (например, `Ivan: 5`).
4. `get_top_players(file_path: str, top_n: int) -> list[str]` **(3 балла)**
   * Читает текстовый файл с рекордами, сортирует игроков по убыванию очков и возвращает список строк вида `Имя - Очки` для топ-`top_n` игроков.

## Вариант 3. Трекер расходов

**Файл решения:** `project_finance.py`

**Сценарий:** Программа хранит все транзакции в файле `expenses.json`. Пользователь может добавлять траты, а программа умеет генерировать красивый текстовый отчет `.txt` по категориям.

**Обязательные функции для реализации:**

1. `load_expenses(file_path: str) -> list[dict]` **(2 балла)**
   * Читает `expenses.json`. Возвращает список словарей: `[{"category": "Еда", "amount": 500.0}]`. Если файла нет или он пустой, возвращает `[]`.
2. `add_expense(expenses: list[dict], category: str, amount: float) -> list[dict]` **(3 балла)**
   * Добавляет новую трату в список словарей и возвращает обновленный список.
3. `save_expenses(expenses: list[dict], file_path: str) -> None` **(2 балла)**
   * Перезаписывает JSON-файл обновленным списком транзакций.
4. `generate_text_report(expenses: list[dict], file_path: str) -> None` **(3 балла)**
   * Группирует траты по категориям, считает сумму по каждой и записывает красивый отчет в текстовый файл.
   * *Пример файла:*
     `Отчет по расходам:`
     `Еда: 1500.0 руб.`
     `Транспорт: 400.0 руб.`
     `Итого: 1900.0 руб.`

## Вариант 4. Шифровальщик текста (Шифр Цезаря)

**Файл решения:** `project_cipher.py`

**Сценарий:** Программа читает секретное сообщение из `secret.txt`, шифрует его с заданным сдвигом, сохраняет зашифрованный текст в `encrypted.txt`, а техническую информацию (метаданные) сохраняет в `meta.json`.

**Обязательные функции для реализации:**

1. `read_text(file_path: str) -> str` **(2 балла)**
   * Читает текстовый файл и возвращает его содержимое единой строкой.
2. `caesar_cipher(text: str, shift: int) -> str` **(4 балла)**
   * Применяет шифр Цезаря к строке `text`. Сдвигает только английские буквы (a-z, A-Z), оставляя знаки препинания и пробелы без изменений. Возвращает зашифрованную строку.
3. `write_text(text: str, file_path: str) -> None` **(2 балла)**
   * Записывает переданную строку `text` в указанный текстовый файл.
4. `save_metadata(original_len: int, shift: int, file_path: str) -> None` **(2 балла)**
   * Создает словарь вида `{"length": original_len, "shift_used": shift}` и сохраняет его в файл JSON.

## Вариант 5. Кассовый аппарат магазина

**Файл решения:** `project_store.py`

**Сценарий:** В файле `catalog.json` хранятся цены товаров. В файле `order.txt` написан список покупок клиента (каждый товар с новой строки). Программа должна посчитать итоговую сумму и выдать чек в файл `receipt.txt`.

**Обязательные функции для реализации:**

1. `load_catalog(file_path: str) -> dict` **(2 балла)**
   * Читает JSON-файл. Ожидается структура `{"Хлеб": 50, "Молоко": 80}`. Возвращает этот словарь.
2. `read_order(file_path: str) -> list[str]` **(2 балла)**
   * Читает текстовый файл со списком покупок (например: `["Хлеб", "Хлеб", "Молоко"]`). Очищает строки от спецсимволов (`\n`). Возвращает список покупок.
3. `process_order(catalog: dict, order: list[str]) -> tuple[float, list[str]]` **(4 балла)**
   * Принимает каталог и список покупок. Подсчитывает общую сумму. Формирует список строк для чека вида: `"Товар - Цена"`. Если товара нет в каталоге, игнорирует его.
   * *Возвращает кортеж* из двух элементов: `(итоговая_сумма, список_строк_чека)`.
4. `generate_receipt(receipt_lines: list[str], total: float, file_path: str) -> None` **(2 балла)**
   * Создает текстовый файл `receipt.txt`. Записывает в него строки из `receipt_lines`, а в конце добавляет строку `Итого: {total}`.
