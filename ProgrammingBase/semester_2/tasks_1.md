# Задание: Функции и структуры данных в Python

## 0. Регламент и критерии оценки

Наша цель — не просто чтобы «код работал», а чтобы он был **профессиональным**.

**Система баллов:**

* **Уровень «Junior» (5 задач):** по 1 баллу (всего 5).
* **Уровень «Middle» (5 задач):** по 3 балла (всего 15).
* **Уровень «Senior» (Мини-проект):** до 10 баллов.
* **Максимум:** 30 баллов.

**⛔ Штрафы за «грязный» код (Clean Code Policy):**
Каждое нарушение снимает **0.5 балла** с задачи.

1. **Невнятный нейминг:** Переменные `a`, `b`, `l`, `d`, `x`. Используйте смысловые имена: `users_list`, `price`, `item`.
2. **Global variables:** Использование глобальных переменных внутри функций запрещено. Всё должно передаваться через аргументы.
3. **Отсутствие Type Hinting:** Аргументы и возвращаемые значения функций должны быть типизированы (например: `def get_sum(nums: list[int]) -> int:`).
4. **Хардкод:** Внутри функций не должно быть `input()` или `print()` (если это не оговорено заданием явно). Функция должна принимать данные и возвращать результат (`return`).
5. **Дублирование кода (DRY):** Если один и тот же кусок логики встречается дважды — вынесите его в отдельную функцию.

## Блок 1. Уровень «Junior» (1 балл за задачу)

*Задачи на проверку базового синтаксиса и простых операций со списками/словарями.*

**Задача 1.1. «Генератор тегов»**
Напишите функцию `create_hashtags(sentence: str) -> list[str]`. Она принимает строку с предложением и возвращает список хэштегов.

* *Пример:* `"Python is awesome"` -> `["#python", "#is", "#awesome"]`
* *Требование:* Все теги должны быть в нижнем регистре.

**Задача 1.2. «Фильтр цен»**
Напишите функцию `filter_prices(prices: list[int], limit: int) -> list[int]`. Она принимает список цен и лимит. Функция должна вернуть новый список, содержащий только те цены, которые строго меньше лимита.

**Задача 1.3. «Перекличка»**
Напишите функцию `cleanup_users(users: list[str]) -> list[str]`. Она принимает список имен, в котором могут быть дубликаты и лишние пробелы по краям (например, `"  Ivan "`).

* *Требование:* Вернуть список уникальных имен, очищенных от пробелов и с заглавной буквы.

**Задача 1.4. «Конвертер оценок»**
Напишите функцию `convert_grades(grades: list[int]) -> dict[str, int]`. Функция принимает список оценок (например, `[5, 4, 3, 5, 2]`).

* *Требование:* Вернуть словарь, где ключом является оценка (строкой: "Отлично", "Хорошо", "Удовл", "Неуд"), а значением — количество таких оценок в списке.
* *Справка:* 5 - Отлично, 4 - Хорошо, 3 - Удовл, 2 - Неуд.

**Задача 1.5. «Поиск сотрудника»**
Напишите функцию `find_phone(phonebook: dict[str, str], name: str) -> str`.

* Принимает словарь (Имя -> Телефон) и имя.

* Если сотрудника нет — возвращает строку `"Не найдено"`.

## Блок 2. Уровень «Middle» (3 балла за задачу)

*Задачи требуют комбинирования логики, циклов и методов коллекций.*

**Задача 2.1. «Слияние складов»**
Есть два словаря, описывающих склады: `warehouse1` и `warehouse2`, где ключ — название товара, а значение — его количество (int).
Напишите функцию `merge_stocks(stock_a: dict, stock_b: dict) -> dict`.

* Функция должна вернуть новый общий словарь.
* Если товар есть на обоих складах, их количество должно суммироваться.
* Если товар только на одном, он просто переносится.

**Задача 2.2. «Аналитика текста»**
Напишите функцию `analyze_text(text: str) -> dict`.
Функция должна подсчитать частоту каждого слова в тексте и вернуть словарь.

* *Условия:* Игнорировать регистр (Слово и слово — одно и то же). Знаки препинания (точки, запятые) не должны быть частью слов.
* *Пример:* `"Привет, мир! Привет"` -> `{'привет': 2, 'мир': 1}`

**Задача 2.3. «Топ-3 студента»**
На вход подается список словарей. Каждый словарь описывает студента: `{'name': 'Ivan', 'scores': [80, 90, 100]}`.
Напишите функцию `get_top_students(students_log: list[dict]) -> list[str]`.

* Функция должна посчитать средний балл каждого студента.
* Вернуть список имен трех студентов с самым высоким средним баллом, отсортированный от лучшего к худшему.

**Задача 2.4. «Валидатор пароля»**
Напишите функцию `check_password_strength(password: str) -> bool`.
Пароль считается надежным (возвращаем `True`), если выполнены ВСЕ условия:

1. Длина >= 8 символов.
2. Содержит хотя бы одну цифру.
3. Содержит хотя бы одну заглавную букву.
4. Содержит хотя бы один спецсимвол из `!@#$%^&*`.
В противном случае вернуть `False`.

**Задача 2.5. «Группировка по возрасту»**
Есть список людей: `[('Anna', 25), ('Max', 18), ('Oleg', 25), ('Kate', 18)]`.
Напишите функцию `group_by_age(people: list[tuple]) -> dict[int, list[str]]`.

* Функция должна сгруппировать людей по возрасту.
* *Результат:* `{25: ['Anna', 'Oleg'], 18: ['Max', 'Kate']}`.

## Блок 3. Уровень «Senior» (Мини-проект, 10 баллов)

**Тема: Консольный менеджер задач (Task Manager)**
Вам нужно реализовать программу для управления списком дел. Хранилище данных — это список словарей. Программа должна работать в вечном цикле, пока пользователь не выберет выход.

**Структура данных (пример одной задачи):**

```python
task = {
    'id': 1,
    'title': 'Купить молоко',
    'priority': 'Высокий',  # или Средний, Низкий
    'is_done': False
}
```

**Требования к функционалу (реализовать в виде отдельных функций):**

1. `add_task(...)`: Добавляет новую задачу. ID должен генерироваться автоматически (быть уникальным).
2. `complete_task(task_id)`: Помечает задачу как выполненную (`is_done = True`).
3. `show_tasks(...)`: Выводит список задач.
    * *Фича:* Должна быть возможность фильтрации: показать все, только выполненные или только активные.
4. `delete_task(task_id)`: Удаляет задачу по ID.

**Требования к интерфейсу:**

* Красивый вывод в консоль (можно использовать f-строки для табличного вида).
* Обработка ошибок: если пользователь вводит ID, которого нет, программа не должна падать, а должна писать «Задача не найдена».

**Критерии оценки проекта (10 баллов):**

* 4 балла: Весь функционал работает корректно.
* 2 балла: Код разбит на функции, есть `main()` блок.
* 2 балла: Грамотная работа со списком (генерация ID, поиск, удаление без ошибок).
* 2 балла: Чистота кода (аннотации типов, docstrings, отсутствие лишних глобальных переменных).

## 3. Правило "Import Guard" (Критично!)

Автоматический тест будет **импортировать** ваши файлы как библиотеки.
Если вы напишете вызов функции или `print()` в основной части файла, этот код выполнится во время теста и сломает проверку.

**Запрещено:**

```python
# junior.py
def create_hashtags(s):
    return ...

# ОШИБКА! Этот код запустится при импорте файла!
print(create_hashtags("Test")) 
```

**Разрешено:**
Используйте конструкцию `if __name__ == "__main__":` для ваших личных проверок. Код внутри этого блока игнорируется при импорте.

```python
# junior.py
def create_hashtags(s):
    return ...

# ПРАВИЛЬНО:
if __name__ == "__main__":
    print("Тестирую функцию:")
    print(create_hashtags("Hello World"))
```

В файле `project.py` весь код запуска меню (цикл `while`, `input`) также должен находиться внутри блока `if __name__ == "__main__":`.
